## Krylov Subspaces

A **Krylov subspace** of order *r* generated by an *m×m* matrix *A* and a vector *v* is the vector subspace spanned by successive matrix-vector products:

$$\mathcal{K}_r(A, v) = \text{span}\{v,\ Av,\ \ldots,\ A^{r-1}v\}$$

Krylov subspaces are the foundation of iterative solvers for large linear systems and eigenvalue problems (GMRES, Lanczos, Arnoldi). Their efficient computation is critical when *m* is large, making them an ideal target for GPU acceleration.

## Algorithm

The benchmark computes a **block Krylov sequence** — a generalization that replaces the starting vector with a matrix *Z*.

### Parameters

| Symbol | Description | Value |
| ------ | ----------- | ----- |
| `m` | Problem size (matrix dimension) | 2¹⁰ ≤ m ≤ 2²⁰ |
| `n` | Block size (constant) | 128 |
| `A` | Square matrix (*m × m*, float) | random |
| `Z` | Block matrix (*m × n*, float) | random |
| `l` | Number of iterations | `l = 2m/n` |

### Loop

For `i = 0, 1, ..., l − 1`:

1. Compute `A · Z` (matrix multiplication, *m×m* × *m×n*)
2. Store the first `n` rows of the result as the snapshot for iteration `i`
3. Update `Z ← A · Z` for the next iteration

The result is a sequence of *l* snapshots of size *n×n*, capturing the evolution of the block Krylov sequence.

## CPU Implementation

The CPU version is written in **plain C with no compiler optimizations** (`-O0`). Its purpose is to establish a performance baseline: correct output, straightforward code, no SIMD, no parallelism, no cache tuning.

### Build & Run

```bash
make              # compile and run with default exponent (EXP=8)
make EXP=10       # run with m = 2^10
make clean        # remove binary
```

Or run directly after compiling:

```bash
echo 10 | ./bench_O0
```

### EXP 10 → 20

```bash
make EXP=10
make EXP=11   
make EXP=12    
make EXP=13    
make EXP=14    
make EXP=15    
make EXP=16  
make EXP=17 
make EXP=18  
make EXP=19  
make EXP=20    
```

### Output

Each run creates a timestamped directory `cpu_{exp}_MMDD_HHMM/` containing:

- `resultado_N.txt` — snapshot of the first *n* rows of *Z* after iteration *N*
- `metricas.txt` — wall time per iteration, total time, peak RSS memory, CPU%, and page faults
